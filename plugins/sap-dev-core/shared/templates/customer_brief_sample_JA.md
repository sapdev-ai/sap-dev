## 6. 品質基準

| 項目 | 値 |
|---|---|
| ABAP Unit テストは必須? | `必須` — 全 public メソッドに最低 1 テスト、ゴールデン I/O が assertion を駆動 |
| ATC は通過必須? | `優先度 1+2 がゲート` (`/sap-atc` で `MAX_PRIORITY = 2`) |
| 最新 ABAP 構文 (`DATA(...)`、`VALUE #( )`)? | `はい` — リリース 7.52 でサポート |
| 新規プログラムの OOP スキャフォールド? | `はい` (lcl_main + run/build/validate/execute/persist) |
| 永続化に変更ドキュメントログ必須? | FI/CO オブジェクトは `はい`、本プロジェクトの MM は `いいえ` (BAPI が処理) |
| メソッドの最大行数? | `50` (既定) — 超えるとチェッカーが `METHOD_TOO_LONG` 警告 |
| 命名プレフィックスの上書き? | `なし` — 既定の `abap_naming_rules.tsv` を使用 |
| ABAP ソースアップロードのファイルエンコーディング? | `UTF-8 BOM なし` (Unicode SAP — codepage 4110) |
| コメント言語 | インラインは `JA OK`、**メソッドヘッダーと例外クラスドキュメントは英語必須** (オフショアレビュアーが読めるように) |

## 7. 設計書ごとの契約 — リマインダー

新しい設計書ごとに、ドキュメント形式 (Word、Excel、PDF) を問わず、
**4 つの枠付きセクション** が含まれていることを確認してください:

1. **インターフェイス契約** — 入力 / 出力 / 例外 (型付き)
2. **検証ルール** — 番号付き、各ルール → `MSG_CLASS-MSGNO`
3. **処理フロー** — 番号付きステップ、各ステップは動詞 1 つ
4. **DDIC オブジェクト** — 名前 / 型 / 桁数 / データ要素の表

仕様書の構造リファレンスは
[`spec_template_JA.xlsx`](spec_template_JA.xlsx) を参照してください。

## 8. 設計書ごとのテストケース — 例

ZHKMM001R01 では、設計書の `黄金テスト` シートに 7 行のゴールデン I/O が
あります (Excel テンプレート参照)。生成器は `ZHKMM001R01_TEST.abap` を
出力し、各行に対して 1 つの `test_*` メソッドを生成します。チェッカーは
ABAP Unit が `assert_*` で 1 件でも失敗するとビルドを失敗させます。

## 各スキルでの使用方法

- `/sap-docs-extract` は設計書 + 本ブリーフを読み、`_PGM_summary.txt`、
  `_process.txt` 等を出力します。
- `/sap-gen-abap` はこのブリーフから MODE フラグを読み、リリース対応の
  最新 ABAP、OOP スキャフォールド、ABAP Unit テスト、依存関係 +
  トレーサビリティファイルを出力します。
- `/sap-check-abap` は品質基準を使って、どの指摘が配備をブロックするかを
  判定します。
- `/sap-atc` は `MAX_PRIORITY = 2` を読み、それに従ってゲートします。

## このサンプルの使い方

1. このファイルをプロジェクト用に `{custom_url}\customer_brief.md` に
   コピーします (注: 言語サフィックスなしのファイル名で配置すれば、
   どの言語環境でも使えます)。
2. HK 固有の値をお客様のプロジェクト値に置き換えます。
3. 該当しない項目は削除するか空白にしてください — 生成器は安全な既定値に
   フォールバックします。
4. リポジトリと一緒にコミットすれば、すべての開発者 + スキルが自動的に
   このブリーフを拾い上げます。
